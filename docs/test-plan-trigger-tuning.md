# 発火トリガー パラメータ設計テストプラン

検知（RMS／スペクトル重心／STT／MediaPipe）のパラメータ設計を、台本化したシナリオで
検証するためのテストプラン。現行パラメータの正は `docs/detection-params.md`。

> **人力実施用の納品物（2026-07-08 作成）**: オペレーター（Biz側）1名＋テスター3名・
> Round 1=第1週／Round 2=翌週の体制で実施するための「テスト設計書（PDF）」と
> 「チェックリスト（Excel）」を `docs/manual-test/` に置いた。現場の実施はそちらが正
> （本書は設計意図と集計・調整の技術リファレンス）。

## 1. 目的と合格基準（ユーザー承認済み・変更禁止）

**目標**: 笑い・怒り等「感情が動いた時」のみ発火し、平常時は発火しない状態にする。

**指標**:
- 再現率（感情イベントへの発火割合）
- 誤発火率（平常シナリオでの発火回数）

**合格基準（Round 2 での判定値）**:

| シナリオ群 | 合格基準 |
| --- | --- |
| 平常会話10分 | 誤発火1回以下 |
| 静寂・朗読・生活音 | 誤発火0回 |
| 大笑い・怒り | 再現率100% |
| 静かな感情（声色のみ・感情ワードのみ） | 再現率50%以上 |

## 2. テスト環境・計測手段

- **実施環境**: 実通話（家族側PC＋高齢者側、クラウド環境）。家族側 `/call?debug=1` のデバッグパネルで
  発火・パラメータ現在値をリアルタイム確認する。
- **計測ログ機能（実装済み・SWA反映済み）**: 1秒ごとのピークホールドサンプル（`rise_peak_db`・`mode`・
  `speech_median_db`・`centroid_ratio_peak`・`speech_ratio` 等）＋全発火イベント（`reason`・
  発火瞬間スナップショット・`photo_count`・`partial_save`）をデバッグパネルの「計測ログDL」ボタンで
  JSONダウンロードする。フィールド仕様は `docs/detection-params.md` の「計測ログ」セクションを正とする。
- **通話画面の常設📊ボタン／ホーム画面からの事後回収（2026-07-08 追加）**: デバッグパネルを開かなくても、
  `/call` 画面左下の常設「📊ログ」ボタン（`NEXT_PUBLIC_MEASUREMENT_UI=1` 時のみ表示）でその場で
  計測ログJSONを即ダウンロードできる。加えて計測ログは通話中10秒ごと＋通話終了時に IndexedDB へ
  自動保存されるため、DLし忘れた場合や画面を閉じてしまった場合でも、家族側ホーム（`/`）の
  「計測ログ（トリガーテスト用）」セクションから直近10通話分を事後にダウンロード・削除できる
  （詳細は `docs/detection-params.md` の「ワンタップDL導線・通話終了後の回収導線」を参照）。
- **計測上の注意（結果解釈時に考慮）**:
  - `speech_ratio` は RMS 用・重心用の観測フレームを合算して算出しており、傾向比較用の相対値として
    使う（絶対的な発話時間割合ではない）。
  - `auto_gain_db` は家族側（自分側マイク）のゲイン値。検知対象である高齢者側ストリームの
    自動ゲインは高齢者側ブラウザで動くためログに含まれない（高齢者側の値が必要なシナリオでは
    高齢者側画面のデバッグ表示を併読する）。
- **役者**: 2名（家族役・高齢者役）。各シナリオは高齢者側マイクで演じる（検知は高齢者ストリームが対象）。
- **記録**: 併せて画面録画を行い、発火通知の出た時刻を正解ラベルと突き合わせる。

## 4. 観点別の検証設計

### 4.1 RMS基準値

- 仮基準→発話基準の切替タイミング（発話累計5秒）の妥当性を検証する。
- `speech_median_db` が役者の平常声の実感と一致するかを確認する。
- A4・A6 で基準が汚染・置き去りにならないかを確認する。

### 4.2 閾値

- 全シナリオの rise 分布を計測ログから取得し、「A系の rise 最大値」と「B系の rise 最小値」の間に
  +12dB（発話基準）が入るかで設計妥当性を判定する。
- ±3dB の感度分析も、取得した分布から机上で再計算する。

### 4.3 スペクトル重心

- 平常会話の `centroid_ratio` 分布と B1/B4 の比を取り、+30% の位置を検証する。
- B4 が RMS で拾えず centroid で拾えることを実証する。

### 4.4 STT

- B5 の命中率と発話→発火遅延を計測する。
- A2 の10分雑談中の感情ワード偶発誤発火頻度を計測する。
- 上記から感情ワードリストの見直し材料を作る。

### 4.5 MediaPipe感情スコア

- B系全発火の連写10枚について、「人間が選ぶベスト順位」と「`face_score` 順位」の一致度（上位一致率）を評価する。
- 無表情ゲート・救済フォールバック（ゲートbypass）の発動記録を確認する。
- 顔角度・逆光の条件別の挙動を観察する。

## 6. C1（咳・くしゃみ）対策の検討枠

C1 は許容ではなく対策検討対象とする。

- Round 1 で咳の音響プロファイル（持続時間・rise・重心比・`speech_ratio`）を計測ログから特徴付ける。
- 対策候補（データを見て選定する）:
  - (a) sustain 延長: 咳が150msより短い破裂音か検証する
  - (b) スパイク棄却: rise が急峻に立って即落ちるパターンを除外する
  - (c) 発火瞬間の MediaPipe 表情ゲート: 無表情・顔なしなら破棄する
  - (d) 重心・スペクトル形状での咳識別
- 各候補の副作用（例: B2「驚き」の取りこぼしリスク）も評価軸に明記する。
