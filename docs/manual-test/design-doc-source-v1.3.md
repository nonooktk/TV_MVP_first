## 1. このテストの目的

このアプリは、テレビ電話の最中に**笑ったり驚いたりした「いい瞬間」を、カメラが自動で写真に残す**仕組みを持っています。

このテストで確かめたいことは、次の2つです。

1. **感情が動いた時（笑う・驚く・怒るなど）に、ちゃんとカメラが反応するか**
2. **ふつうに話しているだけの時に、まちがって反応しないか**

そのために、決められた「台本」どおりに演技をしてもらい、カメラが反応したか・しなかったかを記録します。結果は開発チームがカメラの感度調整に使います。

**Round 1 の結果（お礼と現在地）**: 皆さんの記録のおかげで「誤反応の8割の原因」が特定でき、設定を大きく直せました。Round 2 は「直した結果、ちゃんと静かになったか・拾うべきものは拾えているか」を確かめる回です。

**むずかしい知識は一切いりません。** この設計書のとおりに進めれば、どなたでも実施できます。

### 機器の「ノイズゲート」機能は必ずOFFにする（Round 1 と同じ）

※「ノイズキャンセリング（ノイキャン）」とは別の機能です。ノイキャンは「聞こえる音」を静かにする機能（今回は関係ありません）。機器側のノイズゲートは「マイクに入る小さい音を自動でミュートする」機能で、検知の感度に直接影響します。高齢者役の担当者のパソコン・イヤホン・マイクの設定で「ノイズゲート」「ノイズ抑制」「背景ノイズ除去」などの項目があれば、テスト前に必ずすべてOFFにしてください。

【追加 v1.3】 まぎらわしいのですが、**アプリの中にも「ノイズゲート」という同じ名前の仕組み**が入っています（ごく小さな音を無視する仕組み）。**こちらはアプリの仕様なので、操作は不要です**。OFFにするのは「皆さんの機器側」の設定だけです。

**■ Windows の設定**

1. [システム] ＞ [サウンド] を開き、「入力」で使用中のマイクを選択（またはプロパティ）
2. 「オーディオの強化」（または「オーディオの拡張機能」）があれば「オフ」に変更
3. 【重要】PCメーカー独自のオーディオソフト（Realtek Audio Console／Dell Optimizer／MaxxAudio Pro／HP Audio Control 等）が入っていないか確認し、アプリ内の「マイクのノイズキャンセリング」「環境ノイズ除去」をすべてオフ

**■ Mac の設定**

1. [Appleメニュー] ＞ [システム設定] ＞ [サウンド] ＞ [入力] で使用中のマイクを選択し、「環境ノイズを低減」をオフ（項目が無い機種もあります）
2. 【重要】マイクを使うアプリを立ち上げた状態で、メニューバーの [コントロールセンター] ＞ [マイクモード] を開き、「声を分離」になっていたら「標準」に変更

- 高齢者側（高齢者役）: 同サイトの「待受ページ」（設定済み。開くだけ）

## 5. テスト前の準備手順（毎回、通話を始めるまで）

**全員**: まず、オペレーター・その回の高齢者役・その回の家族役がZoomミーティングに接続する

**家族役（その回の担当者のパソコン）**

1. Chrome を開き、家族側ページを開く
2. 「Googleでサインイン」を押してサインインする
3. 画面左下に「📊ログ」ボタンが見えることを確認する（見えない場合は「8. 困ったときは」へ）
4. 【追加 v1.3】 「📊ログ」ボタンの近くにある**「シナリオ打刻」（番号の選択＋打刻ボタン）**が見えることを確認する
5. 画面録画を開始する
6. チェックリストの「②準備チェック」の項目に✓（オペレーターへZoomで報告）

**高齢者役（その回の担当者のパソコン）**

7. Chrome で待受ページを開く
8. 椅子に座り、画面から50cmくらいの位置で顔が映ることを確認する（テスト中はなるべく同じ距離を保つ）
9. PC・イヤホン・マイクの機器側「ノイズゲート」設定をすべてOFFにする（3章参照）

**通話の開始**

10. 家族役が家族側画面の「通話する」を押す
11. 高齢者役の画面に「でる」ボタンが出るので、高齢者役が押す
12. お互いの顔が映ったら準備完了。オペレーターがZoom上で合図し、台本を上から順に始める

## 6. テスト台本（1組あたり15本・通しで約40分）

### 進め方のルール

- **1組分のテストは、1回の通話の中で通しで行います**（途中で切れたら、かけ直して続きから。**再開したら、まず今の台本番号をもう一度「シナリオ打刻」**してから続ける。メモ欄に「◯番の途中で切断」と記入）。
- 【追加 v1.3・重要】 **各台本の開始手順**: オペレーターがZoom上で「◯番、はじめます」と合図 → **家族役が「シナリオ打刻」で台本番号を選んで打刻** → 演技開始。打刻により、あとの集計が台本ごとに自動化されます（押し忘れても録画から復元できるので、慌てなくて大丈夫です）
- 台本と台本の間は、**15秒くらい静かに待つ**（一度反応したら8秒休む仕組みがあるため）。
- 【追加 v1.3・重要】 **家族役の表情演技のやり方が変わりました**: カメラは「表情が**急に**変わった時」だけ反応する方式になりました（ずっとにこやかでも反応しません）。パート2の各台本では、**直前は真顔に戻しておき、合図とともに一気に表情を作って2〜3秒キープ**してください。じわっと笑うのではなく「パッと変える」のがコツです。
- **反応のサイン**: 反応すると、家族役の画面に「**📸 思い出を記録中…**」→「**思い出を記録しました**」の表示が出ます。家族役はこの表示が出た**回数**を数え、Zoom上でオペレーターへ報告してください（画面共有でもOK）。
- 【追加 v1.3】 反応の表示はこれまでよりコンマ数秒だけ遅れて出ることがあります（咳などの誤反応を見分ける仕組みが入ったため）。異常ではありません。
- 迷ったら「期待どおりか」は空欄でOK。**回数とメモが正確なことが一番大事**です。

### 通話の終わり方（毎回必ず）

1. 全台本が終わったら、**電話を切る前に**家族役が画面左下の「📊ログ」ボタンを押す → 記録ファイルが家族役のパソコンに保存されます
2. ダウンロードされたファイル名（measurement-log-... で始まる）を家族役がオペレーターへ共有し、チェックリストの「④回収ファイル一覧」に書き写す。**通話が複数回に分かれた場合は、どのファイルが何番の台本を含むかもメモ**する
3. 電話を切り、家族役が画面録画を停止して保存する
4. 「📊ログ」を押し忘れて切ってしまっても大丈夫です。ホーム画面の「計測ログ（トリガーテスト用）」一覧から、あとからダウンロードできます

## 7. 記録とファイル回収のやり方

チェックリスト（Excel）の「③実施記録」に、台本1本につき1行記入します。家族役からZoom上で報告を受けて、オペレーターが記入してください。書くのは次の項目です:

1. 実施した日付
2. カメラが反応した回数（0回なら「0」）
3. 期待どおりだったか（プルダウンで ◯ / × を選ぶ。パート1と**C1**は反応0回なら◯、パート2と**C3**は1回以上反応したら◯、C2 は2回以上で◯）
4. 気づいたこと（自由メモ。「2回目の笑いでは反応しなかった」「あくびで反応した」など、なんでも）

**使用環境・マイク機種の記録**（Round 1 と同じ・各担当1回）: 高齢者役の担当者は通話を始める最初に、使用環境（場所・部屋の状況）とマイク／イヤホンの機種を記録してください（チェックリストの「①テスター基本情報」欄）。

**【追加 v1.3】 記録の匿名化**: チェックリスト・ファイル名・フォルダ名には本名やニックネームを書かず、**テスターA/B/C** の表記で統一してください（提出物は開発の公開リポジトリに保管されるため）。

1組分のテストが終わるごとに保存。**最終日（Day 5）に、次の3点セットを開発チームへ送ってください**:

1. チェックリストの Excel ファイル
2. ダウンロードした記録ファイル（measurement-log-...）全部
3. 画面録画の動画ファイル（サイズが大きい場合は相談してください）

## 8. 困ったときは

| 症状 | まず試すこと |
| --- | --- |
| 通話がつながらない | 両方のパソコンでページを開き直す（再読み込み）→ もう一度「通話する」から |
| 「📊ログ」「シナリオ打刻」が見えない | ページを再読み込み。それでも出ない場合は開発チームへ連絡 |
| 相手の声が聞こえない／自分の声が届かない | Chrome の画面上部のマイク・カメラ許可を確認して「許可」を押す |
| シナリオ打刻を押し忘れた | 慌てなくてOK。気づいた時点で今の番号を打刻し、メモ欄に「◯番は打刻漏れ」と記入（録画から復元できます） |
| 「思い出を記録中…」が一度も出ない（パート2でも） | そのまま台本を続けて、メモ欄に「一度も反応せず」と記録（それ自体が大事なデータです） |
| 途中で電話が切れた | かけ直して**今の台本番号を打刻し直してから**再開。メモ欄に「◯番の途中で切断」と記入 |
| 上記で直らない | 無理に進めず、開発チームへ連絡（連絡先: ＿＿＿＿＿＿＿＿＿＿） |

**大事な心構え**: うまく反応しなくても、それは「失敗」ではなく「調整のための大事なデータ」です。ありのままを記録してください。

## 9. 付録: 言葉の言い換え表（開発チームとの連絡用）

ふだんの連絡はぜんぶ左の言葉でOKです。開発チームからの返信に右の言葉が出てきたときの対訳です。

| この設計書の言葉 | 開発チームの言葉 |
| --- | --- |
| カメラが反応する | トリガーが発火する |
| 声の大きさの急な変化 | RMS（音圧）トリガー（Round 2 から基準が +20dB／+26dB に変更） |
| 声の高さ・声色の変化 | スペクトル重心トリガー（**Round 2 では反応オフ・計測のみ**。再設計中） |
