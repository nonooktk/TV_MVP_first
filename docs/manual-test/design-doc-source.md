# テスト設計書 — 「思い出カメラ」反応テスト

TV電話サービス「元気にしてる？」 発火トリガー検証（人力テスト）
版: v1.1 ／ 作成日: 2026-07-08（v1.1 改訂: 同日） ／ 作成: 開発チーム

> **v1.1 の変更点**: カメラが反応する合図が3種類に増えたことに対応しました
> （①おじいちゃん・おばあちゃん側の声 ②家族側の声 ③家族側に映る顔の表情）。
> 台本が15本→18本になり、家族側でも演技する台本（B7・B8・C3）が加わりました。
> 写真は両方の画面から撮られるようになりました（孫と祖父母の両方が残ります）。

## 1. このテストの目的

このアプリは、テレビ電話の最中に**笑ったり驚いたりした「いい瞬間」を、カメラが自動で写真に残す**仕組みを持っています。

カメラが反応する「合図」は3種類あります。

1. **おじいちゃん・おばあちゃん側（テスター側）の声**の盛り上がり（大きな声・声の調子の変化・「すごい」などの言葉）
2. **家族側（オペレーター側）の声**の盛り上がり（歓声など）
3. **家族側の画面に映る顔**の表情（笑顔・変顔）

このテストで確かめたいことは、次の2つです。

1. **感情が動いた時（笑う・驚く・変顔するなど）に、ちゃんとカメラが反応するか**
2. **ふつうに話しているだけの時や、小さな物音では、まちがって反応しないか**

そのために、決められた「台本」どおりに演技をしてもらい、カメラが反応したか・しなかったかを記録します。結果は開発チームがカメラの感度調整に使います。

**むずかしい知識は一切いりません。** この設計書のとおりに進めれば、どなたでも実施できます。

## 2. 体制と役割分担

| 役割 | 人数 | やること | やらなくていいこと |
| --- | --- | --- | --- |
| オペレーター | 1名 | 進行役。家族側パソコンの操作（電話をかける・記録ファイルの保存）、チェックリストへの記入、ファイルの回収。**＋孫役**（一部の台本で、家族側カメラに向かって変顔・歓声の演技をします） | むずかしい設定（初回だけ開発チームと一緒に準備します） |
| テスター | 3名 | 高齢者役。台本どおりに話す・笑う・驚くなどの演技 | パソコンの操作・記録（画面の前に座って演技するだけ） |

- テストは **オペレーターとテスター1名のペア** で行います（テスター3名が交代で同じ台本を演じます）。
- v1.1 から、**オペレーターは「孫役」も兼ねます**（家族側の顔・声もカメラの合図になったため）。台本B7・B8・C3だけなので負担は小さいです。テスト中は自分の顔が家族側カメラに映る位置に座ってください。
- 3名それぞれの声や話し方で試すこと自体が大事なデータになります。**上手な演技である必要はありません。**

## 5. テスト前の準備手順（毎回、通話を始めるまで)

> 初回（第1週 Day 1）だけは、高齢者側パソコンの初期設定が必要です。開発チームと画面を見ながら一緒に行いますので、日程が決まったらご連絡ください。以降は下の手順だけで始められます。

**オペレーター（家族側パソコン）**

1. Chrome を開き、家族側ページを開く
2. 「Googleでサインイン」を押してサインインする
3. 画面左下に「📊ログ」というボタンが見えることを確認する（見えない場合は「8. 困ったときは」へ）
4. 画面録画を開始する
5. チェックリストの「②準備チェック」の項目に✓を付ける

**テスター（高齢者側パソコン）**

6. Chrome で待受ページを開く（オペレーターが事前に開いた状態にしておいてもOK）
7. 椅子に座り、**画面から50cmくらい**の位置で顔が映ることを確認する（テスト中はなるべく同じ距離を保つ）

**オペレーター（追加確認・v1.1）**

7-2. 家族側カメラに**自分（オペレーター）の顔がはっきり映っている**ことを確認する（孫役の台本で使います。逆光・横向きは避ける）

**通話の開始**

8. オペレーターが家族側画面の「通話する」を押す
9. テスター側の画面に「でる」ボタンが出るので、テスターが押す（ここだけ操作があります)
10. お互いの顔が映ったら準備完了。台本を上から順に始める

## 6. テスト台本（1人あたり18本・通しで約45分）

### 進め方のルール

- **1人分のテストは、1回の通話の中で通しで行います**（途中で切れてしまったら、かけ直して続きから。チェックリストのメモ欄に「途中で切断」と書く）。
- オペレーターは各台本の前に「**◯番、はじめます**」と声をかけてから始める（オペレーターの声でカメラは反応しない仕組みですが、もし反応したらメモ欄に書く）。
- **台本と台本の間は、15秒くらい静かに待つ**（カメラには「一度反応したら8秒休む」仕組みがあるため）。
- **反応のサイン**: 反応すると、オペレーターの画面に「**📸 思い出を記録中…**」という表示が出て、少しすると「**思い出を記録しました**」に変わります。オペレーターはこの表示が出た**回数**を数えて記録します。
- **写真は両方の画面から撮られます**（孫側と祖父母側の両方）。あとの写真選び画面には「孫」「祖父母」の目印が付きますが、テスト中に気にする必要はありません。数えるのは上の表示の回数だけです。
- 迷ったら「期待どおりか」は空欄でOK。**回数とメモが正確なことが一番大事**です。

### パート1: ふだんの場面（カメラが反応しない のが正解）

| 番号 | 場面 | テスターがやること（セリフ例） | 時間 |
| --- | --- | --- | --- |
| A1 | 静かにする | 何も話さず、ふつうに座っている。テレビは消す | 2分 |
| A2 | ふつうの雑談 | ふだんの声の大きさで世間話。「今日は暑いねえ」「お昼は そうめんにしようかね」など。オペレーターが相づちを打ちながら続ける | 10分 |
| A3 | 本を読む | 新聞や本を、抑揚をつけずに淡々と音読する | 2分 |
| A4 | 生活音を出す | 話さずに、テレビをつける・食器をカチャカチャさせる・ドアを開け閉めする | 2分 |
| A5 | 最初のあいさつ | （この台本だけ、一度電話を切ってかけ直し）つながった直後に、ふつうの声で「もしもし、こんにちは」 | 30秒 |
| A6 | 立って戻る | 話しながら立ち上がって2〜3歩さがり、また席に戻って話し続ける | 1分 |
| A7 | 小さな物音 | 話さずに、ささやくような小声・紙をめくる音・机を指で軽くたたく音を出す（※v1.1追加: ごく小さな音を無視する仕組みの確認） | 1分 |

### パート2: 感情が動く場面（カメラが反応する のが正解）

| 番号 | 場面 | テスターがやること（セリフ例） | 時間 |
| --- | --- | --- | --- |
| B1 | 声を出して笑う | 大きめの声で「あっはっは！」と笑う | 30秒 |
| B2 | 驚く | 「えーっ！」「うそでしょ！？」と驚く | 30秒 |
| B3 | 怒る | 声を荒げて「だめだって言ったでしょう！」 | 30秒 |
| B4 | 静かに笑う | **声の大きさは変えずに**、「ふふふ」「うふふ」と声の調子だけ上げてくすくす笑う（※少しむずかしいですが、大事な台本です。声を張らないのがコツ） | 30秒 |
| B5 | ほめ言葉 | **ふだんの声のまま**「すごいねえ」「かわいいねえ」と言う | 30秒 |
| B6 | 感極まる | 泣き声まじりで「うれしいねえ…」（泣きまねでOK） | 30秒 |
| B7 | 孫の変顔（※v1.1追加） | テスターは無言のまま。**オペレーターが**家族側カメラに向かって、**声を出さずに**大きな笑顔や変顔をする（口を大きく開ける・ほっぺをふくらませる等） | 30秒 |
| B8 | 家族側の歓声（※v1.1追加） | テスターは無言のまま。**オペレーターが**ふつうの会話のあとに「わあ！すごい！」と歓声をあげる | 30秒 |

### パート3: 境界・連続の場面（C1は記録のみ。C2・C3は「適度な間隔で複数回反応」が正解）

| 番号 | 場面 | やること | 時間 |
| --- | --- | --- | --- |
| C1 | 咳・くしゃみ・あくび | 咳を2〜3回 → 15秒待つ → くしゃみのまね2回 → 15秒待つ → 大きなあくび | 2分 |
| C2 | 笑い続ける | テスターが30秒間、笑い続ける（**2回以上反応するのが正解**。回数を正確に） | 30秒 |
| C3 | 変顔し続ける（※v1.1で内容変更） | テスターは無言のまま。**オペレーターが**30秒間、変顔をし続ける（**2回以上反応するのが正解**。回数を正確に） | 30秒 |

### 通話の終わり方（毎回必ず）

1. 全台本が終わったら、**電話を切る前に**画面左下の「📊ログ」ボタンを押す → 記録ファイルがパソコンに保存されます
2. ダウンロードされたファイル名（`measurement-log-...` で始まる）をチェックリストの「④回収ファイル一覧」に書き写す
3. 電話を切り、画面録画を停止して保存する
4. 万一「📊ログ」を押し忘れて切ってしまっても大丈夫です。ホーム画面の「**計測ログ（トリガーテスト用）**」という一覧から、あとからダウンロードできます

## 7. 記録とファイル回収のやり方

- チェックリスト（Excel）の「③実施記録」に、**台本1本につき1行**記入します。書くのは4つだけ:
  1. 実施した日付
  2. カメラが反応した回数（0回なら「0」）
  3. 期待どおりだったか（プルダウンで ◯ / × を選ぶ。パート1は反応0回なら◯、パート2は1回以上反応したら◯、パート3は C1 のみ記入不要・C2 と C3 は2回以上反応で◯）
  4. 気づいたこと（自由メモ。「2回目の笑いでは反応しなかった」「くしゃみで反応した」など、なんでも）
- テスター1人が終わるごとに保存。**1週の最後（Day 5）に、次の3点セットを開発チームへ送ってください**:
  1. チェックリストの Excel ファイル
  2. ダウンロードした記録ファイル（`measurement-log-...`）全部
  3. 画面録画の動画ファイル（サイズが大きい場合は相談してください）

## 8. 困ったときは

| 症状 | まず試すこと |
| --- | --- |
| 通話がつながらない | 両方のパソコンでページを開き直す（再読み込み）→ もう一度「通話する」から |
| 「📊ログ」ボタンが見えない | ページを再読み込み。それでも出ない場合は開発チームへ連絡（設定が必要な場合があります） |
| 相手の声が聞こえない／自分の声が届かない | Chrome の画面上部にマイク・カメラの許可を求める表示が出ていないか確認し、「許可」を押す |
| 「思い出を記録中…」が一度も出ない（パート2でも） | そのまま台本を続けて、メモ欄に「一度も反応せず」と記録（それ自体が大事なデータです） |
| 途中で電話が切れた | かけ直して続きの台本から再開。メモ欄に「◯番の途中で切断」と記入 |
| 上記で直らない | 無理に進めず、開発チームへ連絡（連絡先: ＿＿＿＿＿＿＿＿＿＿） |

**大事な心構え**: うまく反応しなくても、それは「失敗」ではなく「調整のための大事なデータ」です。ありのままを記録してください。

## 9. 付録: 言葉の言い換え表（開発チームとの連絡用）

ふだんの連絡はぜんぶ左の言葉でOKです。開発チームからの返信に右の言葉が出てきたときの対訳です。

| この設計書の言葉 | 開発チームの言葉 |
| --- | --- |
| カメラが反応する | トリガーが発火する |
| 声の大きさの急な変化 | RMS（音圧）トリガー |
| 声の高さ・声色の変化 | スペクトル重心トリガー |
| ほめ言葉での反応 | STT（音声認識）トリガー |
| 顔の表情での反応（孫の変顔） | 顔トリガー（MediaPipe・face） |
| 家族側の声での反応 | 家族側マイク第2系統（family lane） |
| ごく小さな音を無視する仕組み | ノイズゲート（-50dB） |
| 表情の検出 | MediaPipe（表情スコア） |
| 記録ファイル（📊ログ） | 計測ログ（JSON） |
| 一度反応したら8秒休む仕組み | クールダウン |
| 写真の「孫」「祖父母」の目印 | stream（family / elder） |
